Таблица 5.1 - Протоколы подсистемы IMS

Протокол

Местонахождение и функции

Главные стандарты и спецификации

SIP

Используется для контроля звонков между CSCF и другими элементами сети такими как UE, АС, MGCF.

IETF RFC3261 и продолжение

COPS

Используется для контроля NAT между CSCF и устройствами NAT сети. Он также используется для установки политики между PDF и GGSN, для взаимодействия друг с другом.

IETF RFC2748, RFC3084

Diameter

Используется для обмена информацией между CSCF и HSS или PDF. Он также используется для взаимодействия расчётов между CCF/OCS и CSCF, MRFC, BGCF, MGCF или AS.

IETF RFC3588

ENUM

Используется для изменения номера формата E.164 в доменное имя SIP. Применяется между CSCF и сервером ENUM.

IETF RFC2916

DNS

Используется для изменения доменного имя в IP адрес. Применяется между CSCF/AS и сервером DNS.

IETF RFC1034

H.248

Используется для передачи сообщений управления между MGCF и MGW, а также между MRFC и MRFP.

IETF RFC3525

ITU-T H.248

SNMP

Предназначен для NMS в IMS для взаимодействием с элементами управляемыми.

IETF RFC1157

IPv6

Определяет структуру IP адреса следующего поколения.

IETF RFC2460

IPsec

Используется для защиты безопасности сети между UE и CSCF и между шлюзами безопасности.

IETF RFC2402, RFC2406

SOAP

Используется для обмена информацией между P-CSCF и CLF, между NACF и CLF, и между AS и порталом услуг (SPG). Обмен с использованием протокола расширяемого языка разметки (XML – eXtensible Markup Language) или протокола передачи гипертекста (HTTP – HyperText Transfer Protocol).

W3C (World Wide Web Consortium) SOAP 1.2

NTP

Используется для поддержки синхронизации времени между OMS2600 и сервером NTP. Это позволяет синхронизировать друг с другом все оборудование IMS.

IETF RFC1305

Интерфейс

Местонахождение и функции

Протокол

Gm

Находится между терминалом абонента IMS и P-CSCF. Используется для регистрации и контроля сессии абонента IMS.

SIP

Mw

Используется для обмена сообщениями между CSCF.

SIP

Gq

Находится между P-CSCF и PDF. P-CSCF сообщает PDF о текущей сессии и носителе информации. Таким образом, PDF может выполнить SBLP, сетевое взаимодействие с PDF в других магистральных сетях, а также гарантировать QoS доступа к сетям или межсетевому узлу третьих операторов.

Diameter

Gq'

Находится между P-CSCF и SPDF, и выполняет функции, аналогичные функциям интерфейса Gq.

Diameter

Go

Нспользуется между PDF и GGSN, чтобы отправить политику QoS в плоскости пользователей, и доступа к сети IP для выполнения политики.

COPS

Gx

Находится между PCRF и PCEF. PCRF рассылает инструкции с указанием добавления или удаления политики расчёта. PCEF докладывает, события на каждого пользователя PCRF.

Diameter

Cx

Используется для обмена пользовательской информации между CSCF и HSS:

•вся необходимая информация I-CSCF берётся из S-CSCF

•информация маршрута от CSCF к HSS

•информация, связанная с роуминг авторизацией, CSCF получает от HSS

•параметры безопасности, которые CSCF загружает из HSS необходимые для аутентификации при доступе абонентов IMS

Diameter

Интерфейс

Местонахождение и функции

Протокол

Dx

Находится между CSCF и SLF. Когда оператор IMS имеет несколько HSS баз данных пользователей, интерфейс Dx используется CSCF для того чтобы получить от SLF адрес HSS, который и будет обрабатывать пользовательские данные. Когда оператор имеет один HSS, этот интерфейс не требуется.

Diameter

Mg

Находится между CSCF и MGCF. Через интерфейс, CSCF косвенно контролирует другие сети (не построенные на основе IMS), такие как сети CS, мобильные сети 3G R4 на основе IP, а также фиксированный NGN.

SIP

Mi

Находится между S-CSCF и BGCF. Через интерфейс S-CSCF передает управляющие сигналы для BGCF. Тогда BGCF выбирает MGCF, необходимй для межсетевого взаимодействия с ТфОП и CS 3G/2G сети.

SIP

Mj

Находится между BGCF и MGCF. Через интерфейс, BGCF передает управляющие сигналы для MGCF, когда IMS сеть взаимодействует с ТфОП или МСОП.

SIP

Mk

Находится между BGCF-и. Через интерфейс, BGCF находящийся  в той же сети что и S-CSCF передает управляющие сигналы к BGCF находящийся  в той же сети что и межсетевой узел MGCF.

SIP

Mm

Находится между CSCF и другими внешними не-IMS IP сетями. Через интерфейс, СSCF взаимодействует с другими IMS сетями или не-SIP IMS сетями.

SIP

Mr

Находится между CSCF и MRFC. Через интерфейс S-CSCF получает услуги сетевых ресурсов, таких как медиа-объявления, цифровой набор, конференц-шлюз, и потоковое видео данных.

SIP

Mx

Находится между I-BCF и другими элементами сети такими как CSCF, AGCF и BGCF. Он реализует межсетевое взаимодействие между сетью IMS и другими сетями пакетной коммутации.

SIP

ISC

Находится между S-CSCF и AS. В соответствии с правилами инициирования IMS получает абонентские данные от HSS и запросы SIP отправленные с терминала IMS. S-CSCF решает какую вызвать службу, и затем направляет сессии к конкретной AS для окончательной обработки стоимости услуг.

SIP

Rf

Находится между CCF и другими элементами сети такими как CSCF, MRFC, BGCF и АС. Используется для осуществления сессий связанных с расчётом в offline режиме.

Diameter

Ro

Находится между OCS и другими элементами сети такими как AS, MRFC и OCG. Используется для осуществления сессий связанных с расчётом в online режиме.

Diameter

Rx

Находится между PCRF и AF для обмена информацией на уровне приложений. Информация необходима когда PCRF определяет политику и правила расчёта. PCRF обменивается политикой и правилами контроля расчёта с PCEF.

Diameter

Местонахождение и функции

Протокол

Sh

Находится между HSS и SIP AS или OSA-SCS. Через интерфейс, AS запрашивает HSS, чтобы получить данные, имеющие отношение к логике услуги с добавленной стоимостью и синхронизирует соответствующие данные в HSS.

Diameter

Dh

Находится между SLF и AS или OSA-SCS. В соответствии с данной абонентской информацией и информацией о домашнем домене, AS подтверждает адрес HSS где находятся пользовательские данные.

Diameter

BOSS

Находится между HSS/AS и сервером бизнес-приложений, который используется для передачи соответствующей информационных сообщений.

Simple Object Access Protocol (SOAP)
